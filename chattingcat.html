ChattingCat

PRODUCT DESIGN & FRONT-END DEVELOPEMENT

 

ChattingCat is a platform that connects people who want someone to check their English writing with people who want to help. Through the ChattingCat website or app, writers can conveniently get instant corrections of their work.

As the sole designer, I worked hands-on with users on user experience as well as being involved in strategic product decisions. I also developed all of ChattingCat's web pages using AngularJS.

My work mainly consisted of solving three issues found in both of ChattingCat's website and app. I took on the challenges of how the format only had a single chunky page which was too complex to scale properly, how to effectively create ways for writers and editors to communicate with each other and how to improve proofreading quality and reliability.

Splitting One Chunky Page
to Several Cohesive Pages

PROBLEM

 

ChattingCat started its service with a single web page where writers could request proofreading and a confirmation revision from the editors. However, we kept adding more features to it, which led to one complex chunky page containing all the features that ChattingCat offers. Some important features were overshadowed by other features to the point that it was not providing a good user experience.

SOLUTION

 

I divided one chunky page into several cohesive pages. Each page clarifies what features writers should use in order to personalize their experience while using ChattingCat. The features that had been hidden in the drop down menu in the previous layout are now showcased at the most opportune moment.

Conveying the Context of the Text

PROBLEM

 

ChattingCat started its service with a single web page where writers could request proofreading and a confirmation revision from the editors. However, we kept adding more and more features to it, which led to one complex chunky page containing all the features that ChattingCat offers. Some important features were overshadowed by other features to the point that it was not providing a good user experience.

SOLUTION

 

To avoid any confusion, I determined that the writer should somehow express when the text would be used and what the text is supposed to be about. Two methods were chosen to achieve this: having writers select the purpose of their text and a means for the writers to communicate with the editor.

Conversation between Writers and Editors

PROBLEM

 

After writers receive a refined text from the editors, they were curious about why editors revised it in the way that they did. However, there was not any way to directly ask the editor.

SOLUTION

 

Providing chat boxes below the text ended up being a good solution so that writers could feel free to discuss the changes to their text with the editors.

Improving Editors' Reliability

PROBLEM

 

Writers were often skeptical about whether or not the editors were reliable or qualified enough to revise their writing. Reliability is a very important factor to writers given that ChattingCat is a charged service.

SOLUTION

 

The first action we took was to ask the editors to complete a revision of sample texts and take several proofreading tests before allowing them to become editors. We also decided to share the editors' background through a profile that the editors get to edit themselves.
